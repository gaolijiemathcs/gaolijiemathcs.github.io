map的常用用法详解

发表于 2019-02-17

map翻译为映射，也是常用的STL容器。众所周知，在定义数组时(如int array[100])，其实是定义了一个从int型到int型的映射，例如array[0] = 25、array[4] = 36 就分别是将0映射到25、将0映射到36。一个double类型的数组则是将int类型映射到double型，例如db[0] = 3.14,db[1] = 0.01.但是无论是什么类型，它总是将int型映射到double型。这似乎表现出一个弊端：当需要以其他类型作为关键字来做映射的时候，会显得不太方便。例如有一本字典，上提示了很多的字符串和对应的页码，如果采用数组来表示：“字符串–>页码”这样的对应关系， 就会感觉不太好操作，这时就可以用map，因为map可以将任何类型(包括STL容器)映射到任何的基本类型(包括STL容器)，也就可以建立string类型到int类型的映射。

还可以看一个情况：这次需要判断给定的一些数字在某个文件中是否出现过，按照正常的思路，可以开一个bool型hashTable[max_size]，通过判断hashTable[x]为true还是false来确定x是否在文件中出现，但这就会碰到一个问题，如果这些数字很大(例如有几千位)，那么这个数组就会开不了，这个时候map就会派上用场，因为可以将这些数字当成一些字符串，然后建立string到int的映射(或者是直接建立int到int类型的映射)。

如果还要使用map，这这个时候需要加上map头文件，即#include。除此之外，还需要在头文件下面假设依据:”using namespace std;”,这样就可以在代码中使用map了。下面看一些map的常见用法。

C语言中很多都需要我们自己实现，实现不合适的情况易出错，有些复杂的操作写出来十分的麻烦。C++为使用者提供了标准模板库（Standard Template Library，STL）,其中封装了很多的相当实用的容器，不需要费力的实现他们的细节直接调用函数就可以实现功能，十分的方便。初学的时候不应该被STL的名字吓到，实际上是很容易的东西，学会它对程序的简化有着十分显著的效果

我们首先学习vector的用法。

一般是将1~n这n个整数按某个顺序摆放的结果称为全排列。从递归的角度考虑，如果把问题描述为“输出以1开头的全排列” “输出以2开头的全排列” … “输出以n开头的全排列” 。因此可以不妨设置一个数组P 用来存放当前的排列；再设定一个散列数组hashTable，其中hashTable[x]当整数x已经在数组P当中时为true。

现在按顺序往P的第1位到第n位中填入数字。不妨假设当前已经填写好了P[1]~P[index - 1]正准备填写P[index]。显然需要枚举1~n，如果当前枚举的数字x还没有在P[1]~P[index - 1]中（即hashTable[x] == false)，那么就把它填入P[index]， 同时将hashTable[x]设置为true，然后去处理P的第index+1位(即进行递归)；当递归完成时，再将hashTable[x]还原为false 以便P[index]填写下一个数字

递归边界的问题，就是index达到n+1的时候就说明了P的第1~n位已经填写好了 这个时候就可以将数组P输出，表示生成了一个排列，然后直接return就可以。n=3的时候的全排列生成代码如下：
