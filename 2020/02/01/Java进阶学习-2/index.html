对象的识别

面向对象的思维方式核心：在看到一个东西，每个东西有什么样的属性，这些属性是怎么交互的。

例如设计一个时钟：非面向对象就是两个循环反复 每一分钟+1。

而面向对象的思维方式设计一个时钟，是这个时钟有一些东西组成的。

如果识别和划分一个对象是非常重要的

一个钟的对象划分有不同的方式：

例如一个4位显示器 或者两个两位显示器

显然后者更好，如果划分为两个两位显示器，两者差别仅仅在于跳转回0的上限不同，而基本的操作都相同，就可以吧小时的时钟与分钟的时钟统一为一个类。

对每一个类要求最好相对独立故使用钟联系 hour 和minue

故得到一个时钟模拟的程序

封闭的访问属性

private 只可以用在成员函数(可以是构造函数或者是普通函数) 和成员变量

表示只可以在这个类的内部才可以使用。

私有的变量 在此变量所在类外的地方不可以被.调用

例如在Display类内才可以访问value 和limit

所有的成员变量 应该都是私有的。才可以保护变量不被随意的被使用

只有这个类内部可以访问

类内部指的成员函数和定义初始化

这个限制是对类而不是对对象

同一个类的不同对象 可以访问这个类中不同对象的私有成员！！！！！

所以在分数运算中Fraction的运算当中可以在加法中使用Fraction r中的分子和分母成员变量。

私有是针对类而不是针对对象 好好理解

开放的访问属性

public

任何人都可以访问

任何人指的是在任何类的函数或者定义初始化中可以使用

使用指的是调用、访问或者定义变量

例如在Clock类中可以使用Display类中所有的public函数，

如果没有在前面加上public则在同一个包的其他类可以访问这样叫friendly

一个类的public class 则任何人都可以用这个类定义变量

且每个类的命名必须要与文件的名称相同

编译单元就是一个.java的文件 一个编译单元中可以有很多类

但只有一个类可以作为public 没有public的类指这个类只可以在这个包中起作用 出了这个包就不能用了

一个编译单元只可以有一个public 文件名和类的名字必须相同

包

细节1

可以改为全名

也可以用*代表引用包中的所有类 但是有存在重名引用的风险

所以一般用什么类引用什么类 后在此类中可以直接用引用包中对应引用的类

包是一种管理机制 我们可以在包的下面定义包围display.led 后来吧Display.java放入led中 在文件中就发现放入了display中的文件led中

类变量

static 的成员变量 一个类中有一个类变量。

有两种访问

Display.step

d1.step

但是实际上step只属于Display类 d1 与 d2有办法访问所属Display类中的变量step

输出

static的函数只可以去调用static的函数 只可以去访问static的成员变量

static的函数和成员变量可以通过类的名字去访问，但是也可以用某个对象的名字去访问

这些static的类和对象 不属于任何具体的对象
