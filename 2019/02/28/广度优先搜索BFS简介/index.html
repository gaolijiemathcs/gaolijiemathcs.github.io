广度优先遍历(Breadth First Search, BFS)，一般由队列实现，且总是按照层次的顺序进行遍历，其基本的写法(模板)：

①定义队列q 并且将起点s入队

②写一个while循环，循环条件是队列q非空

③在while循环中，先取出队首元素top，然后访问他(访问可以是任何事情，例如将其输出)。访问完后将其出队。

④将top的下一层结点中所有未曾入队的结点入队，并标记他们的层号为now的层号+1，同时设置这些入队的结点已入过队。

⑤返回②继续循环。

对这个问题的基本求解的基本思想是：枚举每一个位置的元素，如果为0，则跳过；如果为1，则使用BFS查询与该位置元素相邻的4个位置(前提是不出界)，判断他们是否为1(如果某个相邻的位置为1)，则同样去查询与该位置相邻的4个位置，直到整个“1”块访问完毕)。而为了防止走回头路，一般可以设置一个bool型的数组inq(即in queue的简写)来记录每个位置是否在bfs中已入过队。

这里最后还要强调一点，在BFS中设置的inq数组的含义是判断该节点是否已经入过队列，而不是结点是否已经被访问，区别在于，如果设置成是否已经被访问过，有可能在某个结点正在队列中(但是还没有访问)时由于其他节点可以到达这个而将这个再次入队，导致很多结点反复入队，计算量大大增加。因此BFS中让每个结点只入队一次，故需要设置inq数组，判断结点是否入队过 而不是结点是否已经被访问。

最后指出，当使用STL的queue时，元素入队的push操作知识制造了该元素的一个副本入队，因此在入队后对原元素的修改不会影响队列中的副本，而对队列中副本的修改也不会改变原元素，需要注意此处可能会引入bug。
